# 小红书图文生成器 - 设计文档

## 项目概述

将干货长文自动转换为小红书风格的图文内容，包含：
- 封面图（切中主题的标题）
- 4-6张内容图（毒舌但真诚的风格）
- 评论区文案（引导互动）

---

## 一、Prompt 工程

### 1.1 人设定义

```
你是一个毒舌但真诚的朋友
```

**四个核心特质：**

| 特质 | 描述 | 示例 |
|------|------|------|
| 毒舌 | 敢说大实话，不怕得罪人 | "韭菜就是韭菜，说好听点叫什么？" |
| 真诚 | 骂完还给方案，不是为了骂而骂 | 吐槽完问题，紧跟解决方案 |
| 接地气 | 说人话，不端着 | 像闺蜜/哥们儿喝酒时的吐槽 |
| 有见识 | 用行业内幕和真实数据说话 | "$3600"比"很多钱"有说服力 |

### 1.2 语言风格对比

**书面语 vs 口语化：**

| 类型 | 书面语（不要） | 口语化（要） |
|------|----------------|--------------|
| 陈述事实 | "各大云平台已具备一键部署能力，但当时压着不推广" | "云平台早就能一键装了，为什么之前不推？装什么装，无利不起早呗" |
| 数据表达 | "单次任务的Token消耗量是传统对话的30倍以上" | "一个任务烧掉30倍Token，说白了就是个碎钞机" |
| 下判断 | "这可能存在一定风险" | "这就是割韭菜" |

**核心技巧：**
- 用反问代替陈述："为什么？" "装什么装？"
- 用类比代替解释："碎钞机"、"赛博宠物"
- 敢下判断，不模棱两可

### 1.3 区块类型设计

| 类型 | 用途 | 推荐 Label |
|------|------|------------|
| `normal` | 铺垫/背景/细节 | 无（纯陈述） |
| `key` | 扎心真相 | "说白了"、"真相"、"底层逻辑" |
| `bad` | 毒舌吐槽 | "韭菜行为"、"别傻了"、"醒醒" |
| `good` | 正经建议 | "认真的"、"抄作业"、"这样做" |
| `note` | 补刀/彩蛋 | "多嘴一句"、"懂的都懂" |

**每页内容密度：4-6 个区块，避免大片空白**

### 1.4 封面标题要求

- 必须包含文章核心关键词（如"龙虾"、"XX真相"）
- 8-12字，一眼抓住眼球
- 像朋友吐槽时的开场白

## 二、视觉设计规范

### 2.1 配色方案

```python
COLORS = {
    # 基础色
    "bg": "#E3F2FD",            # 浅蓝背景
    "card": "#FFFFFF",          # 白色卡片

    # 文字色
    "title": "#1565C0",         # 深蓝标题
    "title_accent": "#FF6F00",  # 橙色强调（首行）
    "body": "#37474F",          # 深灰正文

    # 装饰色
    "underline": "#FFB300",     # 金黄下划线

    # 区块背景色
    "normal_bg": "#FFFFFF",     # 白色
    "key_bg": "#FFF8E1",        # 淡黄（重点）
    "bad_bg": "#FFEBEE",        # 淡红（警告）
    "good_bg": "#E8F5E9",       # 淡绿（推荐）
    "note_bg": "#E3F2FD",       # 淡蓝（备注）
}
```

### 2.2 布局结构

```
┌─────────────────────────────┐
│         ● ● ●               │  ← 顶部装饰（3个柔和蓝色圆点）
├─────────────────────────────┤
│  ┌─────────────────────┐    │
│  │                     │    │
│  │   标题（橙色首行）   │    │
│  │   ════════          │    │  ← 金黄下划线
│  │                     │    │
│  │   ┌─────────────┐   │    │
│  │   │ ● 区块1     │   │    │  ← 带左侧图标的内容区块
│  │   └─────────────┘   │    │
│  │                     │    │
│  │   ┌─────────────┐   │    │
│  │   │ ★ 区块2     │   │    │
│  │   └─────────────┘   │    │
│  │                     │    │
│  │   ...               │    │
│  │                     │    │
│  └─────────────────────┘    │  ← 白色圆角卡片
│                             │
└─────────────────────────────┘
```

### 2.3 尺寸规范

| 元素 | 尺寸 |
|------|------|
| 图片尺寸 | 1080 x 1440 px（小红书推荐比例） |
| 外边距 | 36px |
| 卡片圆角 | 24px |
| 区块圆角 | 16px |
| 标题字号 | 48px（粗体） |
| 正文字号 | 32px（常规） |
| Label字号 | 28px（粗体） |

### 2.4 区块图标

| 类型 | 图标 | 背景色 |
|------|------|--------|
| normal | ● | #90CAF9（浅蓝） |
| key | ★ | #FF8F00（橙色） |
| bad | ✕ | #E53935（红色） |
| good | ✓ | #43A047（绿色） |
| note | ! | #1E88E5（蓝色） |

### 2.5 装饰元素

**顶部装饰（极简风格）：**
- 3个小圆点，居中排列
- 颜色：#90CAF9（浅蓝）→ #64B5F6（中蓝）→ #90CAF9
- 半径：6px → 8px → 6px

**封面底部装饰（细线风格）：**
- 居中短横线，宽120px，高4px
- 颜色：#90CAF9

## 三、内容结构

### 3.1 输出 JSON 格式

```json
{
  "cover_title": "养龙虾？先问问自己配不配",
  "slides": [
    {
      "title": "为什么突然全民养龙虾？",
      "content": [
        {"type": "normal", "text": "去年11月就出了，极客圈玩了几个月"},
        {"type": "normal", "text": "3月突然爆火，腾讯楼下都在排队装"},
        {"type": "key", "label": "说白了", "text": "Token滞销了，需要新的消耗引擎"},
        {"type": "bad", "label": "别傻了", "text": "这不是技术普惠，是算力去库存"},
        {"type": "note", "label": "懂的都懂", "text": "无利不起早，商业就是商业"}
      ]
    }
  ]
}
```

### 3.2 内容密度要求

- 每页 4-6 个区块
- 动态计算间距，内容自动填满整页
- 避免出现大片空白

## 四、技术实现

### 4.1 核心模块

| 模块 | 职责 |
|------|------|
| `config.py` | 配置加载（API、样式、字体） |
| `llm_formatter.py` | LLM 调用，文章转 JSON |
| `image_generator.py` | PIL 图片渲染 |
| `comment_generator.py` | 评论区文案生成 |
| `main.py` | 主流程编排 |

### 4.2 关键算法

**动态间距计算：**
```python
# 预计算所有区块高度
total_content_h = sum(block_heights)
available_h = bottom_limit - content_start_y

# 动态计算间距
if len(valid_blocks) > 1:
    remaining_space = available_h - total_content_h
    gap = max(16, min(remaining_space // len(valid_blocks), 40))
```

### 4.3 字体配置

- 使用 TTC 字体文件（支持多字重）
- `FONT_INDEX_REGULAR`: 常规字重索引
- `FONT_INDEX_BOLD`: 粗体字重索引
- 跨平台自动检测（见 `font_detector.py`）

## 五、使用方式

### 5.1 命令行

```bash
# 基本用法
python main.py 文章.txt

# 不生成评论区文案
python main.py 文章.txt --no-comments
```

### 5.2 输出结构

```
output/
└── 文章名/
    ├── slide_00_cover.png    # 封面
    ├── slide_01.png          # 内容页1
    ├── slide_02.png          # 内容页2
    ├── ...
    └── comments.txt          # 评论区文案
```
