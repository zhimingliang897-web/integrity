# 爬虫项目综合分析报告

## 一、项目总体概览

当前工作区包含两个独立的爬虫项目，分别是针对**XZ图片网**的图片爬虫和针对**B站（哔哩哔哩）**的视频内容采集工具。以下分别对两个项目进行详细说明。

---

## 二、XZ图片网爬虫项目

### 2.1 项目概述

该项目用于自动化采集XZ图片网（0xzn.com）上的图片图集，支持多分类并行下载、断点续爬和代理轮换功能。

### 2.2 文件结构

```
E:\integrity\7爬虫\xz_图片\
├── xz_config.py      # 配置文件（2.4KB）
├── xz_scraper.py     # 主爬虫程序（48.6KB）
├── xz_proxy.py       # 代理池模块（8.3KB）
├── proxies.txt       # 代理列表文件（如有）
└── data/
    └── xz_images/    # 下载的图片存储目录
        ├── cosplay/
        ├── 机构美女图片/
        └── 网红美女/
```

### 2.3 核心功能

该爬虫具备以下核心功能：

**分类扫描与统计**：程序启动时自动访问目标网站首页，通过多种CSS选择器匹配导航链接，获取全部分类信息，并统计每个分类的图集数量和分页数量。

**交互式选择**：扫描完成后展示分类汇总表格，用户可以通过编号（1、2、3）、范围（1-5）或关键词（all）灵活选择要下载的分类。

**图片批量下载**：自动遍历每个分类下的所有图集页面，通过滚动页面触发懒加载，提取所有图片URL并下载保存到本地目录。

**断点续爬**：每次下载完成后自动保存进度到JSON文件，中断后再次运行可跳过已完成的图集，支持多进程并行运行。

**代理支持**：支持单一代理配置和代理池模式，代理池可从文件加载并自动轮换IP，有效防止单IP请求过于频繁被封禁。

**反检测机制**：使用Playwright浏览器引擎配合stealth模式，模拟真实用户行为，包括随机延迟、随机User-Agent和浏览器指纹。

### 2.4 速度档位配置

| 档位名称 | 页面请求间隔 | 图片请求间隔 | 适用场景 |
|----------|-------------|--------------|----------|
| fast | 1-3秒 | 0.3-0.8秒 | 网络环境好、需要快速采集 |
| normal | 3-6秒 | 0.5-1.5秒 | 默认选项，平衡速度与安全 |
| slow | 5-10秒 | 1-3秒 | 网络一般或目标站点较敏感 |
| safe | 8-15秒 | 2-5秒 | 最大限度降低被检测风险 |

### 2.5 安全限制机制

程序内置多重保护机制以防止对目标站点造成过大压力或被反爬系统拦截：

单次运行请求上限默认为500次，超过后自动停止运行；连续失败3次后暂停60-120秒再继续；每处理20个图集强制休息30-60秒；页面加载超时设置为30秒，图片下载超时设置为60秒。

### 2.6 使用方法

**交互模式（推荐新手使用）**：

运行程序后会自动扫描网站分类并显示汇总表格，按提示输入分类编号即可开始下载。

```bash
cd E:\integrity\7爬虫\xz_图片
python xz_scraper.py
```

**扫描模式（查看分类信息）**：

只扫描分类和数量，不执行下载，便于查看后使用并行命令。

```bash
python xz_scraper.py --scan
```

**指定分类下载**：

通过--category参数指定单个或多个分类，跳过交互界面直接开始下载。

```bash
# 下载单个分类
python xz_scraper.py --category "秀人网"

# 下载多个分类
python xz_scraper.py --category "爱尤物,模范学院"

# 限制每个分类最多下载数量
python xz_scraper.py --category "语画界" --max-albums 100
```

**调整速度与代理**：

```bash
# 使用慢速档位
python xz_scraper.py --speed safe

# 使用代理
python xz_scraper.py --proxy "http://127.0.0.1:7890"

# 清除进度重新开始
python xz_scraper.py --reset-progress
```

**并行多开（高级用户）**：

在多个CMD窗口中分别运行不同分类，可以大幅提升采集效率。

```bash
# 窗口1
python xz_scraper.py --category "秀人网"

# 窗口2
python xz_scraper.py --category "语画界"

# 窗口3
python xz_scraper.py --category "爱尤物,模范学院"
```

## 三、B站爬虫项目

### 3.1 项目概述

该项目是针对B站（哔哩哔哩）视频平台的API调用封装和内容采集工具，支持视频搜索、详情获取、评论抓取和视频下载功能。

### 3.2 文件结构

```
E:\integrity\7爬虫\
├── bilibili_client.py   # API封装核心模块（7.9KB）
├── bilibili_search.py   # 搜索脚本（2.8KB）
├── bilibili_download.py # 下载脚本（3.7KB）
└── data/
    ├── downloads/       # 下载的视频存放目录
    └── [搜索结果目录]/  # 每次搜索创建的独立目录
        ├── search.csv   # 搜索结果表格
        ├── videos.json  # 视频详情和评论
        └── media/       # 下载的封面和视频
```

### 3.3 核心功能

**视频搜索**：通过B站开放API搜索指定关键词的视频，返回标题、作者、播放量、时长等基本信息。

**视频详情获取**：根据BV号获取视频完整信息，包括标题、描述、发布时间、标签、播放统计数据（播放量、点赞、投币、收藏、弹幕数）等。

**评论抓取**：获取视频的热门评论内容，包括用户昵称、评论文本、点赞数和发布时间。

**视频下载**：下载视频的封面图片和视频文件（支持多P视频），可选择视频清晰度。

### 3.4 反封策略

该API封装模块内置以下反封禁策略：

**随机延迟**：每次API请求间隔1-3秒，模拟人类操作节奏，避免请求过于频繁触发风控。

**User-Agent轮换**：维护一个浏览器User-Agent池，每10次请求随机切换一次，降低被识别为机器人的概率。

**匿名Cookie**：启动时自动访问B站首页获取buvid3等必要Cookie，保持Session有效性。

**请求限速**：单次运行最多60次API请求，超过上限后主动停止以保护账号安全。

**文件大小过滤**：视频文件超过100MB时自动跳过，避免下载超大文件导致资源浪费。

### 3.5 使用方法

**搜索视频**：

```bash
cd E:\integrity\7爬虫
python bilibili_search.py "Python教程"
```

搜索完成后会在data目录下创建时间戳命名的文件夹，包含search.csv（搜索结果表格）和videos.json（详细信息和评论）。

**下载视频**：

```bash
# 根据BV号下载
python bilibili_download.py BV1xxx BV2xxx

# 根据搜索结果目录下载（读取videos.json）
python bilibili_download.py data/20260131_153000_Python教程/
```

下载的视频和封面保存在对应的media子目录中。

### 3.6 输出数据格式

**search.csv 表格字段**：

| 字段名 | 说明 |
|--------|------|
| title | 视频标题 |
| bvid | BV号（唯一标识） |
| author | UP主名称 |
| play | 播放量 |
| duration | 视频时长 |
| description | 视频简介 |

**videos.json 详细字段**：

包含视频完整信息：标题、BV号、AV号、CID、作者、描述、时长、各类统计数据（播放、点赞、投币、收藏、弹幕、评论）、发布时间、封面URL、标签数组，以及热门评论列表（top_comments）。

### 3.7 发现的问题

该B站爬虫项目整体代码结构完整，暂无发现明显问题。但存在以下可优化点：

**API稳定性**：依赖B站公开API接口，B站可能随时调整接口参数或认证方式，建议关注接口变化并及时更新代码。

**并发能力**：当前为单线程顺序执行，效率较低，后续可考虑添加多线程/多进程支持。

**错误处理**：部分错误处理较为简单，建议增加更详细的日志记录和重试机制。
