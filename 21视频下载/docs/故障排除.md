## 安装和依赖问题

### 问题1：pip install 失败

**错误信息**：
```
ERROR: Could not find a version that satisfies the requirement yt-dlp
```

**解决方案**：
```bash
# 升级 pip
python -m pip install --upgrade pip

# 重新安装
pip install yt-dlp
```

### 问题2：playwright 安装失败

**错误信息**：
```
playwright install chromium failed
```

**解决方案**：
```bash
# 方法1：使用国内镜像
export PLAYWRIGHT_DOWNLOAD_HOST=https://npmmirror.com/mirrors/playwright/

# 方法2：手动安装
python -m playwright install chromium --with-deps
```

### 问题3：ffmpeg 未找到

**错误信息**：
```
⚠️ ffmpeg 未找到（合并视频流时需要）
```

**解决方案**：

**macOS**：
```bash
brew install ffmpeg
```

**Windows**：
1. 访问 https://ffmpeg.org/download.html
2. 下载 Windows 版本
3. 解压到 C:\ffmpeg
4. 添加到系统环境变量 PATH

**Linux**：
```bash
sudo apt install ffmpeg
```

## 嗅探器问题

### 问题1：浏览器无法打开

**错误信息**：
```
playwright._impl._api_types.Error: Executable doesn't exist
```

**原因**：Playwright 浏览器未安装

**解决方案**：
```bash
playwright install chromium
```

### 问题2：等待很久没有捕获到视频

**可能原因**：
1. 页面加载慢
2. 需要登录但未登录
3. 没有点击播放按钮
4. 视频使用了特殊加密

**解决方案**：

**步骤1**：确保已登录
- 在弹出的浏览器中完成登录
- 等待页面完全加载

**步骤2**：手动点击播放
- 找到视频播放器
- 点击播放按钮
- 等待视频开始缓冲

**步骤3**：检查控制台输出
- 查看是否有 `[🎯 抓到了！]` 提示
- 如果有，说明已捕获成功

**步骤4**：如果仍然失败
- 尝试使用 `ntu_sniffer.py`（NTU专用）
- 或者使用浏览器开发者工具手动提取

### 问题3：捕获到的地址无法下载

**错误信息**：
```
ERROR: Unable to download webpage
```

**原因**：视频地址需要登录凭证

**解决方案**：
```bash
# 1. 先获取 Cookie
python get_cookies.py

# 2. 再运行嗅探器
python sniffer.py
```

### 问题4：浏览器自动关闭

**原因**：等待超时（5分钟）

**解决方案**：
- 修改 `sniffer.py` 中的等待时间
- 找到 `for i in range(300):` 这行
- 将 300 改为更大的数字（如 600 = 10分钟）

## 批量下载器问题

### 问题1：扫描不到任何视频

**可能原因**：
1. 页面结构特殊
2. 视频在 iframe 深层嵌套
3. 需要滚动页面才能加载

**解决方案**：

**方法1**：手动模式
- 脚本会自动进入手动模式
- 在浏览器中手动点击每个视频播放
- 脚本会监听并捕获

**方法2**：修改扫描策略
- 编辑 `batch_downloader.py`
- 在 `scan_page_for_videos` 函数中添加自定义选择器

**方法3**：使用单个嗅探器
- 先用 `sniffer.py` 测试单个视频
- 确认能捕获后再使用批量下载器

### 问题2：某些视频收集失败

**错误信息**：
```
[❌ 未捕获到视频流]
```

**原因**：
1. 视频加载慢
2. 播放按钮未被点击
3. 视频需要特殊权限

**解决方案**：

**步骤1**：查看进度文件
```bash
cat batch_progress.json
# 找到失败的视频编号
```

**步骤2**：单独下载失败的视频
```bash
python sniffer.py
# 粘贴该视频的直接链接
```

**步骤3**：增加等待时间
- 编辑 `batch_downloader.py`
- 找到 `collect_video_urls` 函数
- 增加 `timeout` 参数（默认30秒）

### 问题3：下载速度慢

**原因**：
1. 并发数太低
2. 网络限速
3. 服务器限制

**解决方案**：

**方法1**：增加并发数
```bash
python batch_downloader.py
# 设置并发数为 5
同时下载几个视频？(1-5) [默认: 3]: 5
```

**方法2**：使用代理
- 编辑 `batch_downloader.py`
- 在 yt-dlp 命令中添加 `--proxy` 参数

**方法3**：分批下载
```bash
# 第一批：下载 1-10
python batch_downloader.py
> 选择：2. 选择部分下载
> 输入：1-10

# 第二批：下载 11-20
python batch_downloader.py
> 输入：11-20
```

### 问题4：断点续传不工作

**原因**：进度文件损坏或URL不匹配

**解决方案**：

**步骤1**：检查进度文件
```bash
cat batch_progress.json
# 确认 page_url 是否正确
```

**步骤2**：删除进度文件重新开始
```bash
rm batch_progress.json
python batch_downloader.py
```

**步骤3**：手动编辑进度文件
- 打开 `batch_progress.json`
- 修改失败视频的 `status` 为 `"pending"`
- 保存后重新运行

## 下载问题

### 问题1：下载失败 403 Forbidden

**错误信息**：
```
ERROR: HTTP Error 403: Forbidden
```

**原因**：需要登录凭证

**解决方案**：
```bash
# 1. 获取 Cookie
python get_cookies.py

# 2. 确认 cookies.txt 存在
ls -la cookies.txt

# 3. 重新下载
python main.py
```

### 问题2：视频和音频分离

**错误信息**：
```
WARNING: Requested formats are incompatible for merge
```

**原因**：缺少 ffmpeg

**解决方案**：
```bash
# 安装 ffmpeg
# macOS
brew install ffmpeg

# Windows
# 下载并添加到 PATH

# 验证安装
ffmpeg -version
```

### 问题3：下载速度很慢

**原因**：
1. 网络问题
2. 服务器限速
3. 单线程下载

**解决方案**：

**方法1**：使用批量下载器（并发）
```bash
python batch_downloader.py
# 设置并发数为 3-5
```

**方法2**：使用代理
```bash
# 编辑下载命令，添加代理
--proxy http://127.0.0.1:7890
```

**方法3**：限制画质
```bash
# 编辑 main.py
# 将 "bestvideo+bestaudio/best" 改为 "best[height<=720]"
```

### 问题4：文件名乱码

**原因**：编码问题

**解决方案**：
```bash
# Windows 用户
# 在命令行运行
chcp 65001

# 或者修改文件名模板
# 编辑 main.py
# 将 "%(title)s.%(ext)s" 改为 "video_%(id)s.%(ext)s"
```

## Cookie 问题

### 问题1：无法导出 Cookie

**错误信息**：
```
✗ 未找到浏览器 Cookie 数据库
```

**原因**：
1. 浏览器未安装
2. 路径不正确
3. 浏览器未关闭

**解决方案**：

**步骤1**：确认浏览器已安装
- Chrome、Edge 或 Safari

**步骤2**：完全关闭浏览器
```bash
# macOS
killall "Google Chrome"
killall "Microsoft Edge"

# Windows
# 任务管理器中结束浏览器进程
```

**步骤3**：使用 yt-dlp 内置功能
```bash
python get_cookies.py
# 选择方法1（yt-dlp 内置导出）
# 无需关闭浏览器
```

### 问题2：Cookie 文件无效

**错误信息**：
```
ERROR: Unable to extract video data
```

**原因**：Cookie 过期或格式错误

**解决方案**：
```bash
# 1. 删除旧的 Cookie
rm cookies.txt

# 2. 重新获取
python get_cookies.py

# 3. 确认文件大小
ls -lh cookies.txt
# 应该大于 1KB
```

### 问题3：macOS Keychain 访问被拒绝

**错误信息**：
```
User interaction is not allowed
```

**原因**：需要授权访问 Keychain

**解决方案**：
- 使用 yt-dlp 内置功能（推荐）
- 或者在终端中运行时输入密码授权

## 平台特定问题

### macOS

#### 问题：无法运行 .bat 文件

**原因**：.bat 是 Windows 批处理文件

**解决方案**：
```bash
# 直接运行 Python 脚本
python main.py
python sniffer.py
python batch_downloader.py
```

#### 问题：权限被拒绝

**错误信息**：
```
Permission denied
```

**解决方案**：
```bash
# 添加执行权限
chmod +x main.py
chmod +x sniffer.py
chmod +x batch_downloader.py
```

### Windows

#### 问题：中文乱码

**解决方案**：
```bash
# 在命令行运行
chcp 65001

# 或双击 .bat 文件（已自动设置）
```

#### 问题：找不到 Python

**错误信息**：
```
'python' 不是内部或外部命令
```

**解决方案**：
```bash
# 使用 py 命令
py main.py
py sniffer.py
py batch_downloader.py
```

## 🆘 仍然无法解决？

如果以上方法都无法解决你的问题：

1. **查看详细日志**
   - 运行时添加 `-v` 参数查看详细输出
   - 检查错误信息的具体内容

2. **检查网络连接**
   - 确认能访问目标网站
   - 尝试使用 VPN

3. **更新依赖**
   ```bash
   pip install --upgrade yt-dlp
   pip install --upgrade playwright
   ```

4. **查看 GitHub Issues**
   - yt-dlp: https://github.com/yt-dlp/yt-dlp/issues
   - playwright: https://github.com/microsoft/playwright/issues

5. **手动提取视频地址**
   - 使用浏览器开发者工具（F12）
   - 切换到 Network 标签
   - 筛选 .m3u8 或 .mp4
   - 复制地址后用 `python main.py` 下载

---

**提示**：大多数问题都可以通过重新安装依赖、获取 Cookie 或增加等待时间来解决。
