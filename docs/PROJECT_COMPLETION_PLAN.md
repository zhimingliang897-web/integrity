## 📐 统一设计规范

### 页面结构
每个演示页面应包含：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>项目名称 - Integrity Lab</title>
    <link rel="stylesheet" href="../style.css">
    <link rel="stylesheet" href="../tools-modules/tools-styles.css">
</head>
<body>
    <!-- 导航栏 -->
    <nav>...</nav>
    
    <!-- 页面标题 -->
    <div class="page-header">
        <h1>🎬 项目名称</h1>
        <p>项目简介</p>
    </div>
    
    <!-- 功能演示区 -->
    <div class="demo-section">
        <!-- 演示内容 -->
    </div>
    
    <!-- 使用说明 -->
    <div class="guide-section">
        <!-- 使用步骤 -->
    </div>
    
    <!-- 示例展示 -->
    <div class="example-section">
        <!-- 示例内容 -->
    </div>
    
    <!-- Footer -->
    <footer>...</footer>
</body>
</html>
```

### 目录结构
```
docs/
```

### 样式规范
- 使用全局样式 `style.css`
- 使用工具样式 `tools-modules/tools-styles.css`
- 保持与 tools.html 一致的设计风格
- 响应式布局，支持移动端

### 功能规范
- 每个演示页面都应该是独立的
- 如果需要后端 API，应该复用现有的认证系统
- 提供清晰的使用说明和示例
- 错误提示友好
